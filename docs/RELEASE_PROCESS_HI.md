# रिलीज़, संस्करण और टैगिंग ऑटोमेशन

यह रिपोजिटरी संस्करण टैगिंग और GitHub रिलीज़ को दो वर्कफ़्लोज़ के साथ स्वचालित करता है जो `.github/workflows` में स्थित हैं।

## संस्करण और टैग वर्कफ़्लो

**फ़ाइल:** `.github/workflows/version-and-tag.yml`

**ट्रिगर:** मुख्य में पुश जो निम्नलिखित में किसी भी package.json को बदलता है:
- `frontend/`
- - `frontend/chat-ui/`
  - - `backend/`
   
    - **व्यवहार:**
    - - `frontend/package.json` से विहित संस्करण पढ़ता है
      - - पुष्टि करता है कि यह सिमेंटिक वर्जनिंग (x.y.z) का पालन करता है
        - - अन्य package.json फ़ाइलों में संस्करणों का मिलान सुनिश्चित करता है
          - - यदि टैग v<version> पहले से मौजूद नहीं है, तो एक एनोटेटेड टैग बनाता और धकेलता है
            - - GitHub Actions बॉट आइडेंटिटी का उपयोग करके
             
              - ## रिलीज़ वर्कफ़्लो
             
              - **फ़ाइल:** `.github/workflows/create-release.yml`
             
              - **ट्रिगर:** v*.*.* पैटर्न से मेल खाने वाले टैग के पुश (उदाहरण: v1.2.3)
             
              - **व्यवहार:**
              - - टैग किए गए कमिट पर रिपोजिटरी को चेक आउट करता है
                - - `softprops/action-gh-release` का उपयोग करके GitHub रिलीज़ प्रकाशित करता है
                  - - टैग के बाद रिलीज़ को नामित करता है
                    - - स्वचालित रूप से जनरेट किए गए रिलीज़ नोट्स के साथ
                     
                      - ## GitHub टैग रूलसेट्स
                     
                      - यदि रिपोजिटरी कौन रिलीज़ टैग बना या अपडेट कर सकता है, इसे नियंत्रित करने के लिए टैग रूलसेट का उपयोग करता है, तो सुनिश्चित करें कि रूलसेट `github-actions` बॉट (या जो भी GitHub ऐप/टोकन वर्कफ़्लो को ट्रिगर करता है) को v से शुरू होने वाले टैग बनाने की अनुमति देता है।
                     
                      - एक अनुमति नियम के बिना, स्वचालित नौकरी v*.*.* टैग बनाने का प्रयास करते समय विफल हो जाएगी।
                     
                      - रिपोजिटरी प्रशासक इन सेटिंग्स को **Settings → Rules → Tag rules** के तहत प्रबंधित कर सकते हैं।
                     
                      - ## सर्वोत्तम अभ्यास
                     
                      - ✅ तीन package.json फ़ाइलों में संस्करणों को सिंक रखें
                     
                      - ✅ मुख्य में विलय करने से पहले संस्करण अपडेट करें
                     
                      - ✅ सिमेंटिक वर्जनिंग का पालन करें (MAJOR.MINOR.PATCH)
                     
                      - अधिक जानकारी के लिए मूल दस्तावेज़ देखें: https://github.com/NTARI-RAND/Agrinet/blob/main/docs/release-process.md
